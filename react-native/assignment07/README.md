# Assignment 07 - Data fetching, search, loading, errors

**Estimated time:** ~14h

## Required Video Notes

- `notes/video01.pdf`
- `notes/video02.pdf`

## What You Should Learn

- How to fetch data from a public API
- How to handle loading, empty, and error states
- How to search fetched data
- How to structure a data-driven screen

## Mini Project

Build either a **Movie Browser** or a **Country Browser**.

Recommended APIs:

- [OMDB API](https://www.omdbapi.com/) for movies (free API key
  required, register at omdbapi.com)
- [REST Countries](https://restcountries.com/) for countries (no API
  key needed)

## Points Checklist

- ✅ Data is fetched from a public API: 3 p
- ✅ Loading indicator is shown: 1 p
- ✅ Error state is handled: 2 p
- ✅ Empty result state is handled: 1 p
- ✅ Search field works: 2 p
- ✅ Results are shown in a `FlatList`: 2 p
- ✅ Detail view for one selected item works (e.g., navigate to a
  detail screen or show a modal): 3 p
- ✅ Pull-to-refresh or manual refresh works: 2 p

**Total: 16 p**
