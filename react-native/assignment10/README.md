# Assignment 10 - Final React Native project

**Estimated time:** ~21h

## Required Video Notes

- `notes/video01.pdf`

## What You Should Learn

- How to scope and plan a larger mobile app
- How to combine navigation, state, data, and UI into one project
- How to structure a multi-file React Native codebase
- How to document and present your work clearly

## Final Project

Build one larger React Native application.

Possible topics:

- habit tracker
- course planner
- personal finance tracker
- recipe organizer
- event app
- book finder
- movie browser (must differ substantially from Assignment 07)
- fitness log

## Points Checklist

- ✅ At least 4 screens: 2 p
- ✅ Navigation works across the app: 3 p
- ✅ At least one form with validation: 3 p
- ✅ List rendering with `FlatList`: 2 p
- ✅ Remote data or local persistence (either one satisfies this
  requirement): 3 p
- ✅ Reusable components are used clearly: 2 p
- ✅ Polished styling and consistent UI: 2 p
- ✅ Accessibility is considered: 2 p
- ✅ `README.md` includes screenshots and architecture explanation
  (e.g., folder structure, component tree, or data flow): 2 p
- ✅ Screencast link is included in `README.md`: 1 p
- ✅ App idea is sufficiently large and coherent: 3 p

**Total: 25 p**

### Stronger Projects Usually Also Include

- filtering or search
- meaningful empty/loading/error states
- theme support
- better visual consistency
- clear folder structure
